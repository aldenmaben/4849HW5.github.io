About this app

This is a caculator app that can be controlled by either mouse or keyboard-only inputs.

  • Instructions for using the app
    •

      This calculator app can be controlled on keyboard using the w, s, a, d, and Enter keys. w: up, s: down, a: left, d: right, Enter: select

  • Test results
    •

    • This was a test to determine if it is faster to use a 5(w,s,a,d,enter) button layout or 3(a,d,enter) button layout.

        • Test 1: Compute (((2*7)+4)/3)+9 = 15

        • 5 button: 15.56s, 14.52s, 14.89s; Average:14.99s; Total_Errors:1
        • 3 button: 16.22s, 16.79s, 16.34s; Average16.45s; Total_Errors:4

        Test 2: Compute 2(4+5)-(7+8) = 3

        • 5 button: 12.52s, 12.53s, 12.72s; Average:12.59s; Total_Errors:0
        • 3 button: 21.93s, 20.98s, 20.53s; Average:21.15s; Total_Errors:1

        Test 3: Compute (4((36/2)-9))+7*3 = 57

        • 5 button: 21.77s, 19.82s, 20.12s; Average:20.57s; Total_Errors:3
        • 3 button: 27.46s, 26.87s, 26.64s; Average:26.99s; Total_Errors:4

    • Based off of these results, we can conclude that it is always faster to use 5 buttons than 3 buttons due to the ability of having more functionality. We can also say, that there is a slight learning curve to using only three buttons as when the question difficulty increased, our instinct to find the shortest path(up or down,) caused us to make errors.
